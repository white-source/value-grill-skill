# Value Boardroom Skills（가치 이사회 스킬）

> [English](README.md) | [简体中文](README.zh-CN.md) | [日本語](README.ja.md) | [Español](README.es.md) | [Русский](README.ru.md) | **한국어** | [العربية](README.ar.md)

투자자가 상장 기업을 **소유자이자 운영자**의 시각으로 연구할 수 있도록 돕는 포터블 Agent Skills 라이브러리입니다. 역할 기반의 소크라테스식 이사회 대화를 통해 운영상의 선택을 미래의 잉여현금흐름(FCF)과 연결합니다. 일반적인 투자 보고서나 매매 추천을 생성하는 도구가 아닙니다.

## 목적

- 기업이 현금을 어떻게 **창출하고, 소비하며, 배분하는지** 이해합니다.
- 단기 트레이더가 아닌 이사회 이사 또는 지배 주주로서 사고합니다.
- 검증된 사실을 경영진 주장, 추론, 모델링 가정, 미확인 정보와 분리합니다.
- 부서별 인사이트를 소수의 검증 가능한 FCF 드라이버, 시나리오, 반증으로 변환합니다.

## 설치

두 가지 설치 방법, 두 가지 철학:

- **Skills CLI**: 스킬을 프로젝트에 복사하여 직접 편집하고 나만의 것으로 만들 수 있습니다.
- **Claude Code 플러그인**: 읽기 전용, 항상 최신 상태로 관리되는 번들로 구독하며, 포크하지 않습니다.

### 방법 1 — Skills CLI (편집 가능, 모든 Agent Skills 도구에서 작동)

```bash
npx skills@latest add white-source/value-grill-skill
```

Claude Code, Codex, Cursor, Copilot 및 기타 Agent Skills 호환 도구에서 작동합니다. 파일이 저장소에 복사되며 자유롭게 수정할 수 있습니다.

### 방법 2 — Claude Code 플러그인 (관리형, 플러그 앤 플레이)

직접 유지 관리할 필요 없는 플러그 앤 플레이 설치를 원하시나요? 이 스킬 세트는 Claude Code 네이티브 플러그인으로도 제공됩니다. 편집 가능한 파일을 복사하는 대신, 플러그인이 전체 세트를 관리형 번들로 설치하며 새 버전이 출시되면 업데이트됩니다.

Claude Code 내에서:

```
/plugin marketplace add white-source/value-grill-skill
/plugin install value-grill-skill@white-source
```

또는 셸에서:

```bash
claude plugin marketplace add white-source/value-grill-skill
claude plugin install value-grill-skill@white-source
```

별도의 설정 단계가 필요하지 않습니다. 바로 `/value-boardroom [회사]`로 세션을 시작하세요.

## 시작

```
/value-boardroom [회사명 또는 티커]
```

세션 시작 시 회사(알고 있는 경우 식별자/시장), 기준일, 연구 목적, 증거 모드(사용자 제공만, 또는 에이전트가 접근 가능한 경우 공개 웹 조사), 역할 모드를 확인합니다.

## 역할 반전 — 핵심 메커니즘

이 라이브러리의 시그니처 기능은 **역할 반전**입니다. 이사회 테이블의 양쪽에서 회사를 연구할 수 있으며, 동일한 14개의 운영 역할이 양방향에서 작동합니다.

| 모드 | 모델이 수행 | 사용자가 수행 | 왜 사용하는가 |
|---|---|---|---|
| `briefing` (기본값) | 선택한 경영진 — 인센티브, 예산, 갈등 포함 | 질문하는 이사회 이사 / 파트너 | 실제 운영 세부사항을 추궁하여 이해를 구축; 경영진은 모든 숫자를 현금흐름까지 정당화해야 함 |
| `examination` | 이사회 | 답변하는 CEO, CFO 또는 선택한 경영진 | **당신이** 실제로 비즈니스를 이해하는지 테스트; 이사회는 당신 추론의 허점과 모호함을 포착 |

**briefing** = 질문하며 배우기. **examination** = 질문받으며 배우기. 시작 시 한 번의 질문으로 모드를 선택하고(기본값 `briefing`), 언제든지 `/mode`로 전환 — 또는 그냥 말하세요("역할 바꿔", "나를 시험해봐"). 재시작 불필요. 어느 쪽이든 턴 계약은 동일: 사실 또는 레이블이 붙은 가정을 진술하고, 하나의 인센티브 갈등을 드러내며, 명확한 현금흐름 경로를 추적하고, 정확히 하나의 질문을 합니다.

## 운영 역할

14개의 역할이 회사 전체를 포괄합니다. 자동 완성되는 `/role-<역할>` 바로가기로 하나에 진입하거나(예: `/role-cfo`), `/role ceo cfo ...`로 여러 개를 연결하거나, `/next`로 순차적으로 진행합니다. 각 역할은 자신의 권한 내에서만 발언하며, 예산 또는 전략적 갈등을 표면화하고, 자신의 논점을 수익, 비용, 운전자본, CapEx 또는 자본 배분에 연결합니다.

| 역할 | 담당 | 질문할 내용 |
|---|---|---|
| `secretary` | 이사회 비서 / 코치 — 의제, 질문 트리, 증거, 가정 | 모든 결론에 출처가 있고 추적 가능한지 |
| `ceo` | CEO — 비즈니스 모델, 성장 원천, 전략, 기업 문화, 자본 배분 | 비즈니스 모델 방어력(고객 락인, 전환 비용, 네트워크 효과), 유기적 vs 인수 성장, 재투자율, 문화-인재 정렬, 자본 수익률 |
| `cfo` | CFO — 이익 품질, 레버리지, 운전자본, 배분 | 현금 전환 주기, 발생 vs 현금 이익, 유지 vs 성장 CapEx |
| `product` | 제품 / R&D — 고객 문제, 로드맵, 대체 위험 | 가격 결정력, 유지/이탈, R&D 효율성 |
| `marketing` | 마케팅 / 브랜드 — 획득, 충성도, 광고 의존도 | CAC, LTV, 광고 중단 시 무엇이 먼저 하락하는지 |
| `sales` | 영업 — 수주 품질, 가격, 집중도 | 할인 후 실현 가격, 고객 집중도, 매출채권 |
| `channel` | 채널 — 유통 재고, 머천다이징, 리베이트 | 채널 재고 일수, 판매율, 채널 마진 |
| `supply` | 공급망 / 구매 — 투입 비용, 협상력 | COGS 공급업체 집중도, 계약 조건, 리드타임 |
| `operations` | 운영 / 제조 — 가동률, 수율, 단위 비용 | 가동률, 고정비 흡수, 확장 CapEx |
| `customer-success` | 고객 성공 / 애프터서비스 — 유지, 이탈, 갱신 | 총/순 유지율, 확장 vs 축소, 서비스 비용 |
| `people` | 인사 / 조직 — 핵심 인재, 인센티브, 승계 | 주요 직책 이직률, 인센티브 구조, 병목 |
| `legal-risk` | 법무 / 컴플라이언스 / 리스크 — 라이선스, 소송, 규제 | 우발 부채, 규제 노출, 데이터 거버넌스 |
| `strategy-ma` | 전략 / M&A — 신규 사업, 확장, 인수 | 인수 후 수익률, 영업권, 통합 비용 |
| `ir-pr` | IR / PR — 약속, 투명성, 평판 | 공개 가이던스 vs 실제, 내러티브 vs 운영 |

## 명령어 맵

| 명령어 | 하는 일 |
|---|---|
| `/value-ask [질문 또는 목표]` | 원하는 것을 설명하면 적절한 명령어를 추천하고 설명 |
| `/value-boardroom [회사]` | 회사 이사회 세션 초기화 또는 재설정 |
| `/mode [briefing\|examination]` | 언제든지 역할 모드 전환 — 또는 그냥 말하세요 |
| `/role-<역할>` | **자동 완성**되는 단일 역할 바로가기 — `/role-` 입력 후 선택 (예: `/role-cfo`) |
| `/role [역할...]` | 여러 경영진을 순서대로 연결, 예: `/role ceo cfo supply` |
| `/next` | 제안된 투어의 다음 역할로 이동 — 재입력 불필요 |
| `/research [주제]` | 검증 가능한 공개 정보를 수집 또는 요청하고 증거 클래스로 분류 |
| `/assumption` | 중요한 모델링 가정 보기 또는 수정 |
| `/evidence` | 증거 원장 표시 |
| `/cashflow` | 운영 결과를 수익, 마진, 운전자본, CapEx, FCF, ROIC로 변환 |
| `/stress-test [충격]` | 명명된 충격 실행 또는 현금흐름 체인을 따라 약세/기본/강세 시나리오 구축 |
| `/kill [연도]` | 사전 부검: 선택한 연도까지 현금흐름 실패를 가정하고 역으로 추적 |
| `/board-summary` | 역할극을 종료하고 추적 가능한 이사회 결론을 종합 |

## 예시: Pop Mart (泡泡瑪特, HKEX: 9992)

> 투자 권고가 아닙니다. 모든 주장에는 증거 클래스가 부여됩니다. [전체 워크스루 (중국어) →](docs/examples/popmart-walkthrough.md)

실제 상장 기업과 실제 재무 데이터(Yahoo Finance TTM, Wikipedia)를 이용한 압축 시연입니다.

```text
> /value-boardroom Pop Mart --mode briefing
> /role-cfo
```
| 지표 | TTM (HKD) | 클래스 |
|---|---|---|
| 수익 | 371억 | `Fact` |
| 순이익 | 128억 (34%) | `Fact` |
| FCF | 91억 | `Fact` |
| 현금 | 172억 (D/E 12.6%) | `Fact` |

```text
> /cashflow
```
¥59–69/박스 → 수익 ¥340억 → 고마진 → 경량 운전자본 → CapEx → **FCF HKD 91억**. 3대 드라이버: 동일 점포 트래픽, 해외 ROIC, 프리미엄 카테고리 믹스.

```text
> /stress-test 가격 -10% / > /kill 2030 / > /board-summary
```
충격 전달 체인 + 사전 부검 + 6섹션 이사회 결론 — 모두 **반증 기준이 있는 조건부 스토리**, 단일 예측이 아님. *투자 추천, 목표 주가, 매매 판단 없음.*

## 증거 클래스

모든 주장에 레이블을 부여하여 결론이 추적 가능하도록 유지:

| 클래스 | 의미 |
|---|---|
| `Fact` | 출처가 있는 주장, 발행일과 출처 포함 |
| `Management statement` | 독립적으로 검증되지 않은 회사의 주장 |
| `Inference` | 인용된 사실로부터 도출된 운영상의 결론 |
| `Assumption` | 반증될 수 있는 모델 입력값 |
| `Unknown` | 신뢰할 수 있는 뒷받침이 없는 중요 정보 |

## 웹 조사

가능한 경우 공개 웹 조사를 사용하며 출처와 날짜를 명시합니다. 조사 도구를 사용할 수 없는 경우, 스킬은 **정직하게 성능 저하**됩니다: 누락된 증거를 명시하고, 참조할 1차 출처를 지목하며, 검색어를 제안하고, 다음 이사회 질문을 제시합니다. 데이터, 출처, 접근 권한, 검증을 절대 조작하지 않습니다.

## 안전 경계

이것은 연구 및 학습 프레임워크입니다. 투자 추천, 목표 주가, 매수/매도/보유 신호 또는 거래 지시를 **제공하지 않습니다**.
