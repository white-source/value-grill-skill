# Value Boardroom Skills（Совет директоров）

> [English](README.md) | [简体中文](README.zh-CN.md) | [日本語](README.ja.md) | [Español](README.es.md) | **Русский** | [한국어](README.ko.md) | [العربية](README.ar.md)

Переносимая библиотека Agent Skills, которая помогает инвестору изучать публичную компанию как **владелец и оператор**. Она использует ролевые сократовские диалоги в формате совета директоров, чтобы связать операционные решения с будущим свободным денежным потоком (FCF) — вместо создания универсальных инвестиционных отчётов или торговых рекомендаций.

## Цель

- Понять, как компания **создаёт, потребляет и распределяет** денежные средства.
- Рассуждать как директор совета или контролирующий владелец, а не как краткосрочный трейдер.
- Отделять проверенные факты от заявлений руководства, выводов, модельных допущений и неизвестного.
- Преобразовывать инсайты подразделений в небольшой проверяемый набор драйверов FCF, сценариев и контрдоказательств.

## Установка

Есть два способа установки, две философии:

- **Skills CLI** копирует навыки в ваш проект, чтобы вы могли редактировать их и делать своими.
- **Плагин Claude Code** хранит их как управляемый, только для чтения, всегда актуальный пакет, на который вы подписываетесь, а не форкаете.

### Вариант 1 — Skills CLI (редактируемый, работает с любым инструментом Agent Skills)

```bash
npx skills@latest add white-source/value-grill-skill
```

Совместим с Claude Code, Codex, Cursor, Copilot и другими средами, поддерживающими Agent Skills. Файлы копируются в ваш репозиторий, и вы можете их изменять.

### Вариант 2 — Плагин Claude Code (управляемый, plug-and-play)

Предпочитаете установку "подключил и работает", которую не нужно поддерживать вручную? Этот набор навыков также поставляется как нативный плагин Claude Code. Вместо копирования редактируемых файлов в ваш репозиторий, плагин устанавливает весь набор как управляемый пакет, который обновляется при выходе новой версии.

Внутри Claude Code:

```
/plugin marketplace add white-source/value-grill-skill
/plugin install value-grill-skill@white-source
```

Или из командной строки:

```bash
claude plugin marketplace add white-source/value-grill-skill
claude plugin install value-grill-skill@white-source
```

Отдельный шаг настройки не требуется — начинайте сессию с `/value-boardroom [компания]` сразу.

## Старт

```
/value-boardroom [название компании или тикер]
```

При старте сессия подтверждает компанию (и идентификатор/рынок, если известно), дату отсечения, цель исследования, режим доказательств (только предоставленные пользователем, или публичный веб-поиск, когда агент имеет к нему доступ) и режим роли.

## Инверсия ролей — ключевая механика

Отличительная особенность этой библиотеки — **инверсия ролей**: вы можете изучать компанию с любой стороны стола совета директоров, и одни и те же четырнадцать операционных ролей работают в обоих направлениях.

| Режим | Модель играет | Вы играете | Зачем использовать |
|---|---|---|---|
| `briefing` (по умолчанию) | Выбранного руководителя — с его стимулами, бюджетами и конфликтами | Директора / партнёра, задающего вопросы | Строите понимание, допрашивая реальные операционные детали; руководитель должен обосновать каждую цифру вплоть до денежного потока |
| `examination` | Совет директоров | CEO, CFO или выбранного руководителя — отвечающего | Проверяет, действительно ли **вы** понимаете бизнес; совет ловит пробелы и общие слова в ваших собственных рассуждениях |

**briefing** = учиться, допрашивая. **examination** = учиться, будучи допрошенным. Выбирайте при старте одним вопросом (по умолчанию `briefing`), переключайтесь в любое время через `/mode` — или просто скажите ("reverse roles", "поменяемся ролями"). Без перезапуска. В любом случае контракт хода идентичен: утвердить факт или помеченное допущение, вскрыть один конфликт стимулов, проследить чёткий путь к денежному потоку и задать ровно один вопрос.

## Операционные роли

Четырнадцать ролей охватывают всю компанию. Входите в одну с автодополняемой командой `/role-<роль>` (например, `/role-cfo`), объединяйте несколько через `/role ceo cfo ...`, или проходите пошагово с `/next`. Каждая роль говорит только в рамках своей компетенции, вскрывает бюджетный или стратегический конфликт и связывает свою точку с выручкой, затратами, оборотным капиталом, CapEx или распределением капитала.

| Роль | За что отвечает | Что допрашивать |
|---|---|---|
| `secretary` | Секретарь совета / коуч — повестка, дерево вопросов, доказательства, допущения | Каждый ли вывод имеет источник и прослеживаем |
| `ceo` | CEO — бизнес-модель, источники роста, стратегия, культура, распределение капитала | Защищённость бизнес-модели, органический vs приобретённый рост, норма реинвестирования, соответствие культуры и талантов, доходность на капитал |
| `cfo` | CFO — качество прибыли, леверидж, оборотный капитал, распределение | Цикл конверсии денежных средств, учётная vs денежная прибыль, поддерживающие vs ростовые CapEx |
| `product` | Продукт / R&D — проблема клиента, дорожная карта, риск замещения | Ценовая сила, удержание/отток, эффективность R&D |
| `marketing` | Маркетинг / бренд — привлечение, лояльность, зависимость от рекламы | CAC, LTV, что упадёт первым, если остановить рекламу |
| `sales` | Продажи — качество заказа, ценообразование, концентрация | Реализованная цена после скидок, концентрация клиентов, дебиторка |
| `channel` | Канал — запасы дистрибьюторов, мерчандайзинг, рибейты | Дни запасов в канале, sell-through, маржа канала |
| `supply` | Цепочка поставок / закупки — стоимость входящих ресурсов, рыночная сила | Концентрация поставщиков в себестоимости, условия контрактов, сроки |
| `operations` | Операции / производство — загрузка мощностей, выход годного, удельные затраты | Загрузка мощностей, поглощение постоянных затрат, CapEx расширения |
| `customer-success` | Успех клиента / послепродажное обслуживание — удержание, отток, продления | Валовое/чистое удержание, расширение vs сжатие, стоимость обслуживания |
| `people` | Люди / организация — критический талант, стимулы, преемственность | Текучесть на ключевых должностях, структура стимулов, узкие места |
| `legal-risk` | Юридический / комплаенс / риск — лицензии, судебные разбирательства, регулирование | Условные обязательства, регуляторная подверженность, управление данными |
| `strategy-ma` | Стратегия / M&A — новые направления, экспансия, поглощения | Доходность после поглощений, гудвилл, затраты на интеграцию |
| `ir-pr` | IR / PR — обязательства, прозрачность, репутация | Публичные прогнозы vs факт, нарратив vs операции |

## Карта команд

| Команда | Что делает |
|---|---|
| `/value-ask [вопрос или цель]` | Опишите, что вы хотите; получите правильную команду с объяснением |
| `/value-boardroom [компания]` | Инициализировать или сбросить сессию совета директоров |
| `/mode [briefing\|examination]` | Переключить режим роли в любое время — или просто скажите |
| `/role-<роль>` | Быстрая команда с **автодополнением** — введите `/role-` и выберите (например, `/role-cfo`) |
| `/role [роль...]` | Объединить несколько руководителей в последовательность, например, `/role ceo cfo supply` |
| `/next` | Перейти к следующей роли по рекомендованному маршруту — без повторного ввода |
| `/research [тема]` | Собрать или запросить проверяемую публичную информацию и классифицировать по классу доказательств |
| `/assumption` | Просмотреть или изменить существенные модельные допущения |
| `/evidence` | Показать журнал доказательств |
| `/cashflow` | Преобразовать операционные результаты в выручку, маржу, оборотный капитал, CapEx, FCF, ROIC |
| `/stress-test [шок]` | Запустить именованный шок или построить медвежий/базовый/бычий сценарии по цепочке денежного потока |
| `/kill [год]` | Pre-mortem: предположить сбой денежного потока к выбранному году и пойти назад |
| `/board-summary` | Выйти из ролевой игры и синтезировать прослеживаемое заключение совета |

## Пример: Pop Mart (泡泡瑪特, HKEX: 9992)

> Не является инвестиционной рекомендацией. Каждое утверждение сопровождается классом доказательств. [Полный разбор (на китайском) →](docs/examples/popmart-walkthrough.md)

Показательный прогон на реальной публичной компании с реальными финансовыми данными (Yahoo Finance TTM, Wikipedia).

```text
> /value-boardroom Pop Mart --mode briefing
> /role-cfo
```
| Показатель | TTM (HKD) | Класс |
|---|---|---|
| Выручка | 37,1 млрд | `Fact` |
| Чистая прибыль | 12,8 млрд (34%) | `Fact` |
| FCF | 9,1 млрд | `Fact` |
| Денежные средства | 17,2 млрд (D/E 12,6%) | `Fact` |

```text
> /cashflow
```
¥59–69/коробка → выручка ¥340 млрд → высокая маржа → лёгкий оборотный капитал → CapEx → **FCF HKD 9,1 млрд**. Три драйвера: трафик в сопоставимых магазинах, зарубежный ROIC, премиум-категории.

```text
> /stress-test цена -10% / > /kill 2030 / > /board-summary
```
Цепочка шока + pre-mortem + заключение совета из 6 разделов — всё как **условные истории с критериями фальсификации**, не точечные прогнозы. *Без рекомендаций, целевых цен или торговых инструкций.*

## Классы доказательств

Каждое утверждение помечается, чтобы выводы оставались прослеживаемыми:

| Класс | Значение |
|---|---|
| `Fact` | Утверждение с источником, включая дату публикации и источник |
| `Management statement` | Заявление компании, не проверенное независимо |
| `Inference` | Операционный вывод, сделанный из процитированных фактов |
| `Assumption` | Входные данные модели, которые могут быть опровергнуты |
| `Unknown` | Существенная информация без надёжной поддержки |

## Веб-исследование

Публичный веб-поиск используется, когда доступен, с указанием источника и даты. Когда инструменты исследования недоступны, навыки **честно деградируют**: указывают отсутствующие доказательства, называют первоисточники для изучения, предлагают поисковые запросы и задают следующий вопрос совета. Никогда не выдумывают данные, источники, доступ или проверки.

## Граница безопасности

Это исследовательский и обучающий фреймворк. Он **не** предоставляет инвестиционных рекомендаций, целевых цен, сигналов на покупку/продажу/удержание или торговых инструкций.
