# Value Boardroom Skills（バリュー・ボードルーム・スキル）

> [English](README.md) | [简体中文](README.zh-CN.md) | **日本語** | [Español](README.es.md)

投資家が上場企業を**オーナー・経営者**の視点で研究するための、ポータブルな Agent Skills ライブラリです。役割ベースのソクラテス式ボード（取締役会）対話を通じて、経営上の選択を将来のフリーキャッシュフロー（FCF）に結びつけます。汎用的な投資レポートや売買推奨を生成するためのものではありません。

## 目的

- 企業がどのようにキャッシュを**創り、消費し、配分する**かを理解する。
- 短期的なトレーダーではなく、取締役または支配的オーナーとして思考する。
- 検証済みの事実を、経営陣の発言、推論、モデリングの前提、未知事項と区別する。
- 各部門の知見を、少数の検証可能な FCF ドライバー、シナリオ、反証にまとめる。

## インストール

2 つのインストール方法、2 つの考え方があります：

- **Skills CLI**：スキルをプロジェクトにコピーし、自分で編集して独自化できます。
- **Claude Code プラグイン**：管理された読み取り専用・常に最新のバンドルとして購読し、フォークしません。

### 方法 1 — Skills CLI（編集可能、任意の Agent Skills ハーネスで動作）

```bash
npx skills@latest add white-source/value-grill-skill
```

Claude Code、Codex、Cursor、Copilot、その他の Agent Skills 互換ハーネスで動作します。ファイルはリポジトリにコピーされ、自由に編集できます。

### 方法 2 — Claude Code プラグイン（管理型、プラグアンドプレイ）

手動で保守したくない、すぐ使えるインストールをお望みですか？このスキルセットはネイティブな Claude Code プラグインとしても提供されます。編集可能なファイルをコピーするのではなく、スキル一式を管理バンドルとしてインストールし、新バージョンの公開時に更新されます。

Claude Code 内で：

```
/plugin marketplace add white-source/value-grill-skill
/plugin install value-grill-skill@white-source
```

またはシェルから：

```bash
claude plugin marketplace add white-source/value-grill-skill
claude plugin install value-grill-skill@white-source
```

別途のセットアップ手順は不要です。インストール直後に `/value-boardroom [企業]` で開始できます。

## 開始

```
/value-boardroom [企業名またはティッカー]
```

セッション開始時に、企業（分かれば識別子／市場）、基準日、研究目的、証拠モードを確認します。役割モードは一言の質問で決めるだけ（既定は `briefing`）——フラグ入力は不要です。

## 役割の反転 —— 中核の仕組み

本ライブラリの象徴的な機能は**役割の反転**です。取締役会のテーブルのどちら側からでも企業を研究でき、同じ 14 の経営役割が両方向で機能します。

| モード | モデルが演じる | あなたが演じる | なぜ使うか |
|---|---|---|---|
| `briefing`（既定） | 選択した経営幹部——インセンティブ、予算、葛藤を伴う | 質問する取締役／パートナー | 実際の経営詳細を追及して理解を構築。幹部はすべての数字をキャッシュフローまで説明しなければならない |
| `examination` | ボード（取締役会） | 答える CEO、CFO、または選択した経営幹部 | **あなた自身が**ビジネスを本当に理解しているかを試す。ボードはあなたの論理の飛躍や曖昧さを暴く |

**briefing**＝問いただして学ぶ。**examination**＝問いただされて学ぶ。開始時に一言でモードを選び（既定 `briefing`）、いつでも `/mode` で切り替えられます——「役割を反転して」「私をテストして」と言葉で伝えても OK、再起動不要。どちらでもターンの契約は同じ：事実またはラベル付きの前提を述べ、一つのインセンティブ対立を示し、明確なキャッシュフローの経路を引き、ちょうど一つの質問をすること。

## 経営役割

14 の役割が会社全体をカバーします。自動補完される `/role-<役割>` ショートカットで一つに入る（例：`/role-cfo`）、`/role ceo cfo ...` で複数を連鎖させる、`/next` で順に進めることができます。各役割は自らの所管内でのみ発言し、一つの予算や戦略上の対立を表面化させ、自らの論点を収益・コスト・運転資本・CapEx・資本配分に結びつけます。

| 役割 | 所管 | 問いただすポイント |
|---|---|---|
| `secretary` | ボード秘書／コーチ——議題、質問ツリー、証拠、前提 | すべての結論に出典があり追跡可能か |
| `ceo` | CEO——成長源泉、戦略の境界、資本配分 | 有機的 vs 買収成長、再投資率、投下資本利益率 |
| `cfo` | CFO——利益の質、レバレッジ、運転資本、配分 | キャッシュコンバージョンサイクル、発生 vs 現金利益、維持 vs 成長 CapEx |
| `product` | 製品／R&D——顧客の課題、ロードマップ、代替リスク | 価格決定力、リテンション/チャーン、R&D 効率 |
| `marketing` | マーケティング／ブランド——獲得、ロイヤルティ、広告依存 | CAC、LTV、広告を止めると何が先に落ちるか |
| `sales` | 営業——受注の質、価格、集中度 | 値引き後の実現価格、顧客集中度、売掛金 |
| `channel` | チャネル——販売店在庫、店頭展開、リベート | チャネル在庫日数、売れ行き、チャネルマージン |
| `supply` | サプライチェーン／調達——投入コスト、交渉力 | COGS の仕入先集中度、契約条件、リードタイム |
| `operations` | オペレーション／製造——稼働率、歩留まり、単位コスト | 稼働率、固定費の償却、増設 CapEx |
| `customer-success` | カスタマーサクセス／アフターサービス——リテンション、チャーン、更新 | 粗/純リテンション、拡大 vs 縮小、サービスコスト |
| `people` | 人材／組織——重要な才能、インセンティブ、後継 | 主要ポジションの離職率、インセンティブ構造、ボトルネック |
| `legal-risk` | 法務／コンプライアンス／リスク——許認可、訴訟、規制 | 偶発負債、規制上のエクスポージャー、データガバナンス |
| `strategy-ma` | 戦略／M&A——新規事業、拡大、買収 | 買収後のリターン、のれん、統合コスト |
| `ir-pr` | IR／PR——コミットメント、透明性、評判 | 公開ガイダンス vs 実績、ストーリー vs オペレーション |

## コマンド一覧

## コマンド一覧

| コマンド | 役割 |
|---|---|
| `/value-ask [質問や目的]` | やりたいことを自然言語で伝え、適切なコマンドと解説を提案 |
| `/value-boardroom [企業]` | 企業のボードルーム・セッションを初期化またはリセット |
| `/mode [briefing\|examination]` | いつでも役割モードを切替——「役割を反転して」と伝えるだけでも可 |
| `/role-<役割>` | 単一役割のショートカット、**自動補完**——`/role-` で入力すると一覧から選択（例：`/role-cfo`） |
| `/role [役割...]` | 複数の経営幹部を順に連鎖、例：`/role ceo cfo supply` |
| `/next` | 推奨ツアーの次の役割へ進む——入力し直す必要なし |
| `/research [トピック]` | 検証可能な公開情報を収集・要求し、証拠クラスで分類 |
| `/assumption` | 重要なモデリングの前提を表示・改訂 |
| `/evidence` | 証拠台帳を表示 |
| `/cashflow` | 経営成果を収益・マージン・運転資本・CapEx・FCF・ROICに変換 |
| `/stress-test [ショック]` | 指名ショックを実行、またはキャッシュフロー連鎖でベア／ベース／ブル案件を構築 |
| `/kill [年]` | プレモーテム：選択した年までにキャッシュフローが破綻すると仮定し、遡って原因を探る |
| `/board-summary` | ロールプレイを終了し、追跡可能なボード結論を統合 |

## 証拠クラス

すべての主張にラベルを付け、結論が追跡可能な状態を保ちます：

| クラス | 意味 |
|---|---|
| `Fact` | 出典付きの主張。出版日と出典を含む |
| `Management statement` | 独立的に検証されていない企業の主張 |
| `Inference` | 引用された事実から導かれた経営上の結論 |
| `Assumption` | 反証可能なモデル入力 |
| `Unknown` | 信頼できる裏付けのない重要な情報 |

## ウェブ研究

利用可能な場合は公開ウェブによる研究を用い、出典と日付を明記します。研究ツールが利用できない場合は、**正直にグレードダウン**します：欠落している証拠を明示し、参照すべき一次情報を挙げ、検索案を提案し、次のボード質問を提示します。データ、出典、アクセス、検証を捏造することは決してありません。

## 安全上の境界

これは研究・学習フレームワークです。投資推奨、目標株価、売買・保有の判断、取引指示は**提供しません**。

## 検証

```bash
npm test
```

Node 組み込みのテストランナー（`node --test tests/skills.test.mjs`）を実行し、スキルのレイアウト、フロントマター、共有リファレンス、安全契約を検証します。Node 20+ が必要です。
